## 1.1, 리액트 개요

* React(리액트): 사용자 인터페이스(UI, User Interface)를 구축하기 위한 자바스크립트 라이브러리. 2013년 페이스북(현 메타)에서 처음 공개되었으며, 현재는 가장 널리 사용하는 프런트엔드 라이브러리.

- 리액트는 ***선언적***이며, ***컴포넌트 기반의 접근 방식***을 제시.
- '모든 UI를 컴포넌트로 바라본다'는 리액트의 관점은 복잡한 웹 애플리케이션을 구조적이고 체계적으로 설계할 수 있는 방향을 제시.

* 리액트의 설계 철학
  - 컴포넌트 기반 아키텍처
  - 가상 DOM
  - 선언적 프로그래밍 : UI를 만들 때 '무엇을 보여줄지(what)'를 선언하는 방식
    - 명령형 프로그래밍(imperative programming) : UI를 만들 때 '어떻게 변경할지(how)'를 작성하는 방식
    - 선언적 프로그래밍(declarative programming)

- 렌더링(rendering): 화면에 표시할 내용을 생성하고 그리는 과정

* 리액트의 특징
  - 단방향 데이터 흐름(one-way data binding)
    - 데이터 흐름을 예측하기 쉬움
    - 상태 변화를 추적하기 용이함
    - 대규모 애플리케이션에서도 복잡성을 줄일 수 있음
    - 애플리케이션의 동작을 명확히 이해할 수 있음
  - JSX 사용
  - 상태 관리 도구 제공
    - 훅(hook)은 함수형 컴포넌트에서 상태를 관리하고, 생명주기(lifecycle)와 관련한 기능을 사용할 수 있도록 하는 리액트의 내장 기능.
  - 풍부한 생태계
    - Next.js와 Gatsby
    - React Native : 하나의 코드베이스로 iOS와 Android 애플리케이션을 동시에 개발할 수 있는 모바일 프레임워크
    - 상태 관리 라이브러리
    - UI 컴포넌트 라이브러리
