## 3.3, 컴포넌트의 기초.

* 컴포넌트(Component)란:
  - UI를 반환하는 함수(또는 클래스)
  - JSX를 반환하고, props를 입력으로 받아 UI를 결정하는 단위
  * 컴포넌트 = UI를 설명하는 함수

  * 컴포넌트로 인식되는 조건
    1. 함수 이름이 대문자로 시작
    2. JSX를 반환
    3. 리액트 렌더 트리에서 호출됨

- 파스칼 케이스(PascalCase)란 각 단어의 첫 글자를 대문자로 쓰고, 공백이나 구분자 없이 이어 붙이는 표기법.

- react 서버 실행법
  ```
  npm install
  npm run dev
  ```

- 다른 파일에 있는 컴포넌트를 사용할 때 import 문을 사용
  ```tsx
  import 컴포넌트_이름 from '파일_경로';
  ```
  ```tsx
  ./
  상대 경로에서 ./는 현재 파일이 위치한 폴더
  ../는 상위 폴더(한 단계 위)를 나타냄
  파일 경로에서 확장자(.tsx)는 생략 가능
  ```

- 컴포넌트 트리(component tree)
- App을 **루트 컴포넌트(root component)**라고 함.
- 리액트에서는 컴포넌트가 어디에서 렌더링되는지에 따라 컴포넌트 트리에서의 위치가 결정됨.

* 리액트 컴포넌트의 주요 특징
  - 계층 구조
  - 단방향 데이터 흐름(props, 3.4절에서 참고)
  - 재사용성
  - 상태 관리
  - 렌더링 최적화

## 3.4, 컴포넌트와 props 객체

- props(properties)는 컴포넌트를 마치 HTML 태그처럼 사용해 값을 속성 형태로 전달함.
- 리액트에서는 부모 컴포넌트가 자식 컴포넌트에 모든 속성을 하나의 객체로 모아서 자식 컴포넌트로 전달. 이 객체를 **props 객체**라고 함.

- 구조 분해 할당(destructuring assignment): 객체나 배열의 속성을 추출해 변후에 할당하는 자바스크립트 문법

* 함수 시그니처(Function Signature)란? 이 함수를 어떻게 호출해야 하는지 한눈에 보여주는 정보

- 타입스크립트의 인터페이스(interface)
- .d.ts
 
- 전개 연산자(spread operator)
- 전개 연산자를 사용하면 코드가 간결해지고 필요한 속성만 선택적으로 전달할 수 있어 편리함.
  
- [x] 구조 분해 할당. 113쪽 이해 안감. (이해 완료.)

* children은 컴포넌트 태그 사이에 작성된 모든 내용을 하나로 묶어 전달하는 특별한 속성.
