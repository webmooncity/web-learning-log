# 5장, 컴포넌트 스타일링

## 5.1 전통적인 방법으로 스타일링하기

- 전통적인 CSS 파일을 활용하는 방식, CSS-in-JS, Tailwind CSS

### 5.1.1, 인라인 스타일

* **인라인 스타일**(inline style)
  : JSX에서는 스타일을 객체 형태로 작성. 이때 속성 이름은 카멜 케이스로 작성하고, 값은 문자열로 지정.
  `<h1 style={{ color: 'red', fontSize: '20px' }}>h1</h1>;`

  - 첫 번째 예제: 스타일 객체를 변수로 분리해서 사용하는 방식.
  - 두 번째 예제: JSX 안에서 style 속성에 직접 객체를 작성하는 방식.

### 5.1.2, 글로벌 스타일

- ***글로벌 스타일***(global style): .css 확장자를 가진 파일에 CSS 코드를 작성하고, 이를 컴포넌트에서 import 해 적용하는 방식. 일반적으로 외부 스타일이라고 함.
- 글로벌 스타일은 특정 컴포넌트에만 국한되지 않고, 애플리케이션 전체에 영향을 미침.

### 5.1.3, CSS 모듈

- **CSS 모듈**(CSS module)
- CSS 모듈의 가장 큰 특징은 스타일이 **로컬 스코프**(local scope)

### 5.1.4, classnames 라이브러리

* 해시(hash): 어떤 값을 넣으면, 규칙적으로 만들어지는 짧은 코드(문자열)
  - 원본을 대표하는 '지문(fingerprint)' 같은 문자열을 만들어내는 걸 해시라고 부름
- 암호화: 되돌릴 수 있음(키로 복호화)
- 해시: 보통 되돌리기 어렵게 만들고 "요악, 지문'으로 씀
