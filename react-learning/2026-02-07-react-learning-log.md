## 10.2, useEffect 훅

* **사이드 이펙트**(side effect)란? JSX를 렌더링하는 본래 목적 외에 발생하는 부수적인 행동을 의미함.
- 리액트에서는 사이드 이펙트를 처리하기 위해 **useEffect** 훅을 사용함.

- useEffect 훅은 다음과 같은 세 가지 시점에서 사이드 이펙트를 처리할 수 있음.
  * 마운트(mount, 생성): 컴포넌트가 처음 생성되어 화면에 나타날 때.
  * 업데이트(update, 수정): 상태나 props가 변경되어 컴포넌트가 리렌더링될 때
  * 언마운트(unmount,소멸): 컴포넌트가 화면에서 사라질 때

- 형식 `useEffect(setup, dependencies)`
  - setup(설정 함수)
  - dependencies(의존성 배열)
    - 만약 의존성 배열을 생략하면 컴포넌트가 리렌더링될 때마다 useEffect 훅이 실행됨.
    - Object.is

- **컴포넌트의 생명주기**(lifecycle): 리액트의 모든 컴포넌트는 마운트(생성) -> 업데이트(수정) -> 언마운트(소멸) 과정을 거침.

- 리액트는 상태가 바뀌면 해당 컴포넌트를 리렌더링하며, 이 과정을 업데이트라고 판단함.

### 10.2.2, useEffect 훅 사례.

- useEffect 훅을 활용하는 대표적인 사례
  - API 호출
  - 타이머 설정
  - 실시간 이벤트 처리하기
  - 자동 저장 기능 구현하기
  - 실시간 통신 기능 구현하기
    : 실시간 기능이 필요한 웹 애플리케이션에선느 웹소켓(WebSocket)을 사용해 서버와의 양방향 통신을 구현하는 경우가 많음.
